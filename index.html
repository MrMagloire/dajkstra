An illustration of the shortest-path problem

This tool illustrates finding the shortest path in a graph in two ways:

  • The naive strategy enumerates all possible paths from the initial node (in green) to the final node (in red).
  • The Dijkstra strategy incrementally builds a tree of shortest-paths from the initial node (in green) using Dijkstra's shortest-path algorithm.
The naive strategy might appear reasonable but it does not scale. Use the Hard Map button to get a map with many paths and compare the naive enumeration to Dijkstra's solution.

The code requires HTML Canvas support in your browser. If it does not work, try using the Google Chrome browser with which the code has been tested.

— — —

Disclaimer: A visualization step does not accurately reflect a computational step of the algorithm. While the depiction of the naive enumeration is inaccurate to the actual search through the tree of possible paths (i.e., the naive search does share path prefixes) it does illustrate the potential size of the search space. The visualization of Dijkstra's shortest-path algorithm shows a single step as the assignment of costs to nodes adjacent to the current minimal-cost node. However, there are several computational steps in order to find this minimal-cost node.
